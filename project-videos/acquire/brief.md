# Project Video Brief

Project: Acquire

Video type: Plain overview video

Target duration: 41-42 seconds

Primary viewer: Hiring manager or general portfolio viewer

Story mode: Plain overview

One-sentence job: Acquire turns scattered opportunity tracking into one governed local workflow for sourcing, review, applications, outreach, evidence, and next actions.

Viewer takeaway: This is a working operating system for a messy business workflow, not a generic productivity dashboard.

Plain story:

- Problem: opportunity tracking was split across job boards, inboxes, notes, drafts, browser tabs, and follow-up lists.
- Solution: one local command center shows status, blockers, and the next action for each opportunity.
- Result: every pursuit has a clearer next action, approval state, history, and evidence trail.

Scene arc:

1. Problem: opportunities were spread across too many tools.
2. Cost: the next step was hard to trust.
3. Solution: Acquire shows the workflow, status, and next action in one local command center.
4. How it works: judgment and approval gates come before automation.
5. Result: each pursuit keeps its materials, answers, status, blockers, and evidence together.
6. Why it matters: this is business-systems engineering in miniature.

Reading-speed constraint:

- Use one headline plus one short sentence at most.
- Keep scenes around 14-15 words and hold each for roughly 6.5 seconds.
- Reduce visible copy when the viewer needs to inspect the product screenshot.
- Keep stack names and command output off-screen.

Assets needed:

- Screenshot: public Acquire Pursuits command-center screenshot from the portfolio case study.
- UI clip: not needed; motion is built in HyperFrames around the static proof surface.
- Artifact: native cards for sources, reviews, approvals, evidence, and next action.
- Diagram/data point: simple governed workflow rail, not technical architecture.

Privacy notes:

- Hide: local database content, real applications, resumes, outreach drafts, company-specific answers, personal profile data, private browser/session state, tokens, cookies, and local secret paths.
- Safe to show: the public case-study screenshot, public project name, plain workflow labels, and synthetic/cards-only proof elements built for the video.
