# Project Video Brief

Project: Agent Recall

Video type: Technical proof video

Target duration: 42 seconds

Primary viewer: Technical reviewer or hiring manager checking implementation depth

Story mode: Evidence tour

One-sentence job: Agent Recall turns local agent transcript archives into searchable memory records with source receipts, freshness checks, and reusable context.

Viewer takeaway: Agents need memory they can inspect and reuse, not unverifiable summaries.

Plain story:

- Problem: useful agent work was buried in local transcript archives.
- Solution: Agent Recall keeps raw archives local, builds a disposable search index, and extracts receipt-backed memory records.
- Result: prior decisions, commands, blockers, and facts become searchable, inspectable, and reusable by another agent.

Scene arc:

1. Problem: agents need memory they can inspect and reuse.
2. Input: local session archives remain the source of truth.
3. Action: Agent Recall redacts, parses, and indexes high-signal events.
4. Durable output: memories and search results carry receipts.
5. Proof: doctor and watcher states make freshness visible.
6. Result: prior work becomes reusable with an inspectable trail.

Reading-speed constraint:

- Use one headline plus one short sentence at most.
- Keep each technical scene below 15 must-read words.
- Hold each scene for roughly 6.5 seconds.
- Reserve the final 3 seconds for the settled end card.

Assets needed:

- Screenshot: public Agent Recall CLI screenshot from the portfolio case study.
- UI clip: not needed; motion is built in HyperFrames around the static proof surface.
- Artifact: native cards for archives, parser, redaction, index, receipts, memory records, and freshness.
- Diagram/data point: simple local-source-to-receipt pipeline.

Privacy notes:

- Hide: private transcript content, private memory records, private local paths beyond already-public portfolio-safe examples, tokens, cookies, API keys, and secret-bearing command output.
- Safe to show: the public case-study screenshot, public project name, public GitHub URL, public release metadata, tool-level labels, and synthetic proof cards.
