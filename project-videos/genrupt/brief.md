# Project Video Brief

Project: Genrupt

Video type: Flagship AI systems video

Target duration: 45 seconds

Primary viewer: Hiring manager or general portfolio viewer

Story mode: Plain overview with visible product proof

One-sentence job: Genrupt turns AI media generation into a commercial Amazon seller workflow platform with paid-team foundations, recoverable jobs, and safe agent access.

Viewer takeaway: This is production AI systems work that made a fast-moving product commercially operable for seller teams.

Plain story:

- Problem: Genrupt had AI media working, but paid seller teams needed more than generation screens.
- Solution: the product gained commercial foundations, recoverable long-running jobs, and a narrow agent workflow surface.
- Result: the platform supported the first 100 paying customers and made newer seller workflows safer to ship.

Scene arc:

1. Problem: AI media was working, but seller teams needed a real operating platform.
2. Cost: each new workflow made billing, credits, retries, and recovery more important.
3. Solution: Genrupt became a seller workflow platform with one shared foundation.
4. Commercial foundation: subscriptions, credits, billing, and account state supported paid teams.
5. Agent access: allowed workflows could run through a safer product surface.
6. Result: seller workflows became product surfaces, and the platform supported the first 100 paying customers.

Reading-speed constraint:

- Use one headline plus one short sentence at most.
- Keep implementation terms off-screen.
- Hold product screenshots long enough for the viewer to understand the proof.
- Reserve the final 3 seconds for a settled end card.

Assets needed:

- Screenshot: public Genrupt projects surface.
- Screenshot: public Genrupt dashboard and account overview.
- Screenshot: subscriptions and pricing surface.
- Screenshot: Agent Access surface, redacted so provider labels are not enlarged.
- Screenshot: A+ project setup surface.
- Screenshot: market-analysis surface.
- Artifact: native cards for billing, credits, jobs, allowed workflows, status, and result.

Privacy notes:

- Hide: account names, profile photos, private account footer data, provider names where not needed, routes, IDs, job mechanics, secrets, tokens, cookies, and file paths.
- Safe to show: public portfolio screenshots after redaction, public project name, plain workflow labels, and native proof cards.
