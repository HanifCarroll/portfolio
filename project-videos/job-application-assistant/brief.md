# Project Video Brief

Project: Job Application Assistant

Video type: Plain overview video

Target duration: 40-41 seconds

Primary viewer: Hiring manager or general portfolio viewer

Story mode: Plain overview

One-sentence job: Job Application Assistant turns a job posting into a reviewed draft, saved PDF, and application record.

Viewer takeaway: This is a small product that makes a real job-search workflow more organized.

Plain story:

- Problem: applying to jobs created too many loose pieces.
- Solution: one local assistant keeps the job, draft, PDF, and history together.
- Result: each application is easier to review, save, and follow up on.

Scene arc:

1. Problem: application work was scattered.
2. Cost: it was easy to lose track of what was sent or saved.
3. Solution: one local workflow brings the pieces together.
4. How it works: review, draft, export, and track.
5. Result: every application has a clear saved record.
6. Why it matters: the project turns a messy workflow into a product.

Reading-speed constraint:

- Use one headline plus one short sentence at most.
- Keep most scenes below 15 must-read words.
- Hold each scene for roughly 6-7 seconds.
- Reduce text when the viewer also needs to inspect product UI.

Assets needed:

- Screenshot: synthetic product surface showing side panel plus dashboard.
- UI clip: not needed for the pilot; motion is handled in HyperFrames around a static surface.
- Artifact: synthetic PDF card.
- Diagram/data point: simple connected steps, not technical architecture.

Privacy notes:

- Hide: real job applications, real companies from runtime data, private cover letters, emails, phone numbers, file paths with secrets, and personal employment data.
- Safe to show: synthetic job/company data, public project name, plain feature labels, and synthetic PDF/history cards.
