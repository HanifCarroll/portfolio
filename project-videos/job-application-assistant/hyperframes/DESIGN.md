# Design

## Style Prompt

Use a quiet product-systems style that matches the portfolio and the Job Application Assistant interface: soft off-white surfaces, ink text, forest-green accents, table and ledger motifs, and restrained motion. The copy should be short, plain, and written for a nontechnical viewer. Structure the video as a simple overview: problem, cost, solution, how it works, result, and why it matters.

## Colors

- Canvas: `#eef3ee`
- Panel: `#fbfcf9`
- Ink: `#14211c`
- Muted text: `#52625a`
- Border: `#d5ded6`
- Accent: `#1f6f55`
- Accent soft: `#dfeee7`
- Secondary accent: `#b56b2a`
- Dark proof panel: `#18251f`

## Typography

- Display: `Georgia`, serif
- Interface/body: `Avenir Next`, `Helvetica Neue`, sans-serif
- Data/code: `SF Mono`, `Menlo`, monospace

## Motion

- Tight explainer pacing.
- Entrances use varied direction, opacity, scale, and rule reveals.
- Primary transitions are push slides; topic changes use blur crossfades.
- Keep the product surface readable for at least six seconds.
- Keep text readable: prefer fewer words and longer holds over fast copy.
- Avoid technical proof scenes; the viewer should understand the project without knowing the implementation stack.

## What NOT to Do

- Do not use neon, purple-blue gradients, or generic dark SaaS styling.
- Do not show private job/application data.
- Do not make the video depend on decorative title cards only.
- Do not put schema names, API names, test output, or command output on screen.
- Do not use jump cuts between scenes.
- Do not use oversized web-hero composition when the product screen should carry the proof.
