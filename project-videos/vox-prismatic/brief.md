# Project Video Brief

Project: Vox Prismatic

Video type: Short workflow demo

Target duration: 33 seconds

Primary viewer: Hiring manager or general portfolio viewer

Story mode: Short demo with plain overview language

One-sentence job: Vox Prismatic turns long transcripts into reviewable post drafts and a human-led scheduling workflow.

Viewer takeaway: This is a content operations workflow that turns long inputs into posts someone can review, shape, and schedule.

Plain story:

- Problem: useful ideas were buried in long transcripts.
- Solution: one workflow extracts moments, drafts posts, supports hook review, and keeps publishing human-led.
- Result: raw material becomes reviewable, scheduled output instead of disappearing in a transcript.

Scene arc:

1. Problem: long transcripts buried good ideas.
2. Cost: drafting posts by hand was slow and voice-sensitive.
3. Solution: Vox Prismatic turns transcripts into reviewable drafts.
4. Workflow: extract ideas, shape posts, review hooks, and schedule.
5. Result: long inputs become posts someone can trust.
6. Ending beat: Vox Prismatic, reviewable posts from long transcripts.

Reading-speed constraint:

- Use one headline plus one short sentence at most.
- Keep every scene below 15 must-read words.
- Hold each proof surface long enough for the viewer to inspect the product.
- Keep stack names, provider names, schema names, and command output off-screen.

Assets needed:

- Screenshot: cropped posts workspace from the public Vox Prismatic hero image.
- Screenshot: cropped hook workbench from the public Vox Prismatic feature image.
- UI clip: not needed; motion is built in HyperFrames around static product surfaces.
- Artifact: native cards for transcript, insight, draft, review, and scheduled post states.
- Diagram/data point: simple workflow rail, not technical architecture.

Privacy notes:

- Hide: account footer text, private email, runtime account details, tokens, cookies, local paths, and any unpublished transcript data.
- Safe to show: cropped public portfolio screenshots, public project name, plain workflow labels, and native synthetic cards built for the video.
